# Спринт 3 — Полировка и портфолио-фичи

Продолжительность: 2 недели. Цель — довести продукт до состояния «показа на собеседовании»: безопасность, деплой, бонусные фичи.

## Цели

- Закрыть вопросы прав доступа, импорта/экспорта, вложений.
- Подготовить прод-окружение и CI/CD.
- Добавить «вау»-фичи для демонстрации (SSE, отчёты, интеграции).

## Задачи по ролям

### Backend

1. RBAC: роли `ADMIN/USER`, granular permissions в DRF.
2. Вложения для задач/сделок (S3/MinIO + presigned URL).
3. CSV/Excel экспорт клиентов/сделок.
4. Webhooks или интеграции (например, уведомления в Slack/Telegram).
5. Улучшение мониторинга: health-check, `/metrics`, алерты.

### Frontend/BFF

1. SSE/notifications (подписка на Redis pub/sub, отображение в UI).
2. Улучшенный dashboard (виджеты, графики, сохранённые фильтры).
3. Drag-n-drop Kanban (HTMX + JS helper).
4. Модальные формы, inline-редактирование клиентов/сделок.
5. Документация сценариев демонстрации (скрипты для презентации).

### DevOps / Общее

1. `docker-compose.prod.yml`, Gunicorn конфиги, nginx TLS.
2. CI/CD pipeline (GitHub Actions) с тестами и линтерами.
3. Примеры `.env.prod`, инструкция деплоя (README раздел).
4. Автоматизация seed-данных (`make seed`, management commands).
5. Финальный код-ревью и подготовка записей демо-видео.

## Definition of Done

- Проект запускается в prod-режиме (reverse proxy, TLS, Gunicorn).
- RBAC и безопасность проверены (тесты + ручные сценарии).
- Присутствуют «показательные» фичи (вложения, экспорт, уведомления).
- Документация готова для демонстрации: README, plan, sprint-ноты, скрипты.
